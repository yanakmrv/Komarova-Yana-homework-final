# Автоматизированный мониторинг и реагирование на угрозы

## Описание проекта
Данный проект представляет собой инструмент на Python для автоматического анализа потенциальных угроз информационной безопасности. Скрипт выполняет следующие задачи:
- Сбор данных из двух источников: логов сетевой активности (Suricata) и API VirusTotal.
- Анализ IP-адресов из логов на предмет вредоносной активности.
- Имитация реагирования (блокировка подозрительных IP).
- Формирование отчёта в формате CSV.
- Визуализация результатов в виде графика (топ IP по количеству детектов).

Проект выполнен в рамках итогового домашнего задания по дисциплине «Программирование на Python».

## Используемые источники данных
1. **Логи Suricata** — файл `suricata_logs.json` в формате JSON Lines, содержащий события сетевой безопасности (поля: timestamp, src_ip, dest_ip, proto, app_proto).
2. **API VirusTotal** — публичное API для проверки репутации IP-адресов. Используется для получения статистики обнаружений (malicious, suspicious, harmless, undetected).

## Требования к окружению
- Python версии 3.8 или выше.
- Установленные библиотеки: `requests`, `pandas`, `matplotlib`.

## Установка и запуск
### 1. Клонирование репозитория
```bash
git clone https://github.com/your_username/threat-monitoring.git
cd threat-monitoring
```

### 2. Установка зависимостей
```bash
pip install requests pandas matplotlib
```

### 3. Получение API-ключа VirusTotal
- Зарегистрируйтесь на [virustotal.com](https://www.virustotal.com).
- Перейдите в раздел API и скопируйте ваш ключ.
- Вставьте ключ в файл `threat_monitor.py` в переменную `API_KEY` (замените `"ВАШ_API_КЛЮЧ_VIRUSTOTAL"`).

### 4. Подготовка файла с логами
В директории проекта должен находиться файл `suricata_logs.json` с примером логов. Пример содержимого:
```json
{"timestamp":"2025-03-10T10:00:01.123456+0000","src_ip":"8.8.8.8","dest_ip":"192.168.1.1","proto":"TCP","app_proto":"dns"}
{"timestamp":"2025-03-10T10:00:02.234567+0000","src_ip":"1.1.1.1","dest_ip":"192.168.1.1","proto":"TCP","app_proto":"http"}
{"timestamp":"2025-03-10T10:00:03.345678+0000","src_ip":"185.130.5.133","dest_ip":"192.168.1.1","proto":"TCP","app_proto":"ssl"}
{"timestamp":"2025-03-10T10:00:04.456789+0000","src_ip":"91.240.118.77","dest_ip":"192.168.1.1","proto":"TCP","app_proto":"http"}
{"timestamp":"2025-03-10T10:00:05.567890+0000","src_ip":"45.155.205.233","dest_ip":"192.168.1.1","proto":"TCP","app_proto":"dns"}
```

### 5. Запуск скрипта
```bash
python threat_monitor.py
```
Скрипт последовательно опрашивает VirusTotal для каждого уникального IP из логов (с задержкой 15 секунд между запросами для соблюдения ограничений бесплатного тарифа). После завершения анализа будут созданы файлы отчёта и графика.

## Результаты работы
- **`threat_report.csv`** — таблица с результатами проверки IP, содержащая колонки: ip, malicious, suspicious, harmless, undetected, country, timestamp.
- **`threat_graph.png`** — горизонтальная столбчатая диаграмма (топ-10 IP по количеству вредоносных детектов).

## Примечания
- Для избежания превышения лимитов API VirusTotal установлена задержка 15 секунд между запросами. При необходимости её можно увеличить.
- В бесплатной версии VirusTotal количество запросов ограничено (до 4 в минуту). Если вы планируете проверять много IP, рекомендуется увеличить задержку или приобрести платный доступ.
- Скрипт имитирует реагирование на угрозы только выводом сообщений в консоль. Для реальной блокировки потребуется интеграция с системами защиты (например, iptables, API брандмауэра).

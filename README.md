# 🛡️ Автоматизированный мониторинг и реагирование на угрозы

![Python](https://img.shields.io/badge/python-3.8%2B-blue)
![License](https://img.shields.io/badge/license-MIT-green)

Инструмент на Python для автоматического анализа потенциальных угроз информационной безопасности. Скрипт собирает данные из логов сетевой активности (Suricata) и проверяет IP-адреса через VirusTotal API, имитирует реагирование (блокировку подозрительных адресов) и формирует отчёт с визуализацией.

## Описание проекта
Проект разработан в рамках итогового домашнего задания по дисциплине «Программирование на Python». Он демонстрирует комплексный подход к автоматизации задач информационной безопасности: сбор логов, обогащение данных через внешние сервисы, принятие решений и генерация отчётов.

## Возможности
- **Сбор данных** из двух источников:
  - Логи сетевой активности (формат JSON Lines, Suricata).
  - API VirusTotal для проверки репутации IP-адресов.
- **Анализ** IP-адресов из логов на предмет вредоносной активности.
- **Имитация реагирования**: блокировка подозрительных IP (выводится в отчёт).
- **Формирование отчёта** в формате CSV с детальной информацией по каждому IP.
- **Визуализация** результатов: построение графика топ-10 IP по количеству обнаружений (malicious + suspicious).

## Используемые источники данных
1. **Логи Suricata**  
   - Файл: `suricata_logs.json` в формате [JSON Lines](https://jsonlines.org/).  
   - Содержит события сетевой безопасности с полями: `timestamp`, `src_ip`, `dest_ip`, `proto`, `app_proto`.

2. **API VirusTotal**  
   - Публичное [REST API](https://developers.virustotal.com/reference) для проверки репутации IP-адресов.  
   - Возвращает статистику обнаружений: `malicious`, `suspicious`, `harmless`, `undetected`.

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/your_username/threat-monitoring.git
cd threat-monitoring
```

### 2. Получение API-ключа VirusTotal
1. Зарегистрируйтесь на [virustotal.com](https://www.virustotal.com).
2. Перейдите в раздел [API](https://www.virustotal.com/gui/user/your_username/apikey) и скопируйте ваш ключ.
3. Откройте файл `threat_monitor.py` и вставьте ключ в переменную `API_KEY` (замените `"ВАШ_API_КЛЮЧ_VIRUSTOTAL"`).

### 3. Подготовка данных
Создайте файл `suricata_logs.json` в корне проекта и наполните его тестовыми событиями в формате JSON Lines. Пример содержимого:
```json
{"timestamp":"2025-03-10T10:00:01.123456+0000","src_ip":"8.8.8.8","dest_ip":"192.168.1.1","proto":"TCP","app_proto":"dns"}
{"timestamp":"2025-03-10T10:00:02.234567+0000","src_ip":"1.1.1.1","dest_ip":"192.168.1.1","proto":"TCP","app_proto":"http"}
{"timestamp":"2025-03-10T10:00:03.345678+0000","src_ip":"185.130.5.133","dest_ip":"192.168.1.1","proto":"TCP","app_proto":"ssl"}
{"timestamp":"2025-03-10T10:00:04.456789+0000","src_ip":"91.240.118.77","dest_ip":"192.168.1.1","proto":"TCP","app_proto":"http"}
{"timestamp":"2025-03-10T10:00:05.567890+0000","src_ip":"45.155.205.233","dest_ip":"192.168.1.1","proto":"TCP","app_proto":"dns"}
```

### 4. Запуск скрипта
```bash
python threat_monitor.py
```

> **Важно:** Бесплатный тариф VirusTotal ограничивает количество запросов (4 запроса в минуту). Скрипт автоматически добавляет задержку 15 секунд между вызовами API.

После выполнения в папке проекта появятся:
- `threat_report.csv` — отчёт с результатами проверки.
- `top_detections.png` — график топ-10 IP по количеству обнаружений.

## Примечания
- Скрипт обрабатывает **только уникальные IP-адреса** из логов, чтобы не превысить лимиты API.
- Имитация блокировки подозрительных IP означает, что в отчёте помечаются IP с `malicious > 0` или `suspicious > 0` (при необходимости логику можно изменить).
- Для реального применения потребуется адаптировать скрипт под конкретную инфраструктуру (например, интеграция с файрволом).
